Object oriented programming
¶

  • Programs are collections of individual units: the objects
  • Objects have distinct types and (hence) purposes
  • Functionalities of disctinct types have limited (or no) implementation dependencies
  • An object gathers data (state) and functions (behavior)

wikipedia: Object oriented (OO) programming

wikipedia: OO Fundamental concepts

Classes and Objects¶

  • A Class is a type definition, constructed as the blue print used to create objects

  • A Class defines

    • Members : variables describing object states. Also called data members, characteristics, attributes, or properties.
    • Methods : the actions associated with a particular object.
    • Constructors : particular methods setting the initial state
    • Destructors : particular methods destroying the object and freeing memory
  • An Object is an instance (allocated in memory) constructed from a Class description

Cat example¶

class Cat:
   # the constructor
   def __init__ (self, name):
      # the member name
      self.name = name

   #A given method
   def scream(self):
      print('MIAOU')

   # one accessor
   def getName(self):
      return self.name

self stands for the current object or instance

Python constraints and features¶

  • Methods have unique names
  • named parameters are accepted
class Cat:
   def scream(self, scream='MIAOU'):
     print(scream)
  • Constructor with arguments (if construction needs arguments)
class Cat:
   def __init__(self, name='kitty'):
    self.name = name
  • destructor have no other argument than self
class Cat:
   def __del__(self):
    print('I am dying')

Inheritance¶

  • When a Class is based on another Class
  • Re-use/Factor implementation/code
  • getName is an accessor (why is it necessary ?) appearing twice
  • Cat's scream is different from Dog's scream

Python inheritance syntax¶

class Dog(Animal):

    # the constructor needs to call the parent one
    def __init__ (self, **kwargs):
       Animal.__init__(self, **kwargs)

    def __del__(self):
       Animal.__del__(self)

    def scream(self):
        print('OUAFF')
  • scream is simply overwritten (dictionary fashion) $\Rightarrow$ Overloading/Polymorphism

Python inheritance syntax¶

class Dog(Animal):

    # the constructor needs to call the parent one
    def __init__ (self, *args, **kwargs):
       super().__init__(*args, **kwargs)

    def __del__(self):
       super().__del__()

    def scream(self):
        print('OUAFF')
  • super() calls in a generic way to the mother class

Abstract Class¶

from abc import ABC, abstractmethod

class Animal(ABC):

item getter/setter & python operators¶

class MyClass:

  def __init__(self):
    self.mylist = [ 1, 2, 3, 4]
  def __getitem__(self, index):
    return self.mylist[index]
  def __setitem__(self, index, value):
    self.mylist[index] = value
  def __delitem__(self, index):
    del self.mylist[index]
  • Try it !
obj = MyClass()
print(obj[2])

obj[3] = 4
del obj[0]

Exceptions¶

  • Making and raising your own exceptions
class MyError(RuntimeError):
  def __init__(self, value):
    self.parameter = value
  def __str__(self):
    return repr(self.parameter)

  raise MyError(['important info', 1234])
